Big Mechanism

Big mechanisms are large, explanatory models of complicated systems in which interactions have important causal effects. The collection of big data is increasingly automated, but the creation of big mechanisms remains a human endeavor made increasingly difficult by the fragmentation and distribution of knowledge. The goal of the Big Mechanism program is for machines to help humans to model and analyze very complicated systems by reading fragmented literatures and assembling the reasoning with models.

Harvard Medical School INDRA INDRA (the Integrated Network and Dynamical Reasoning Assembler) assembles information about biochemical mechanisms into a common format that can be used to build several different kinds of explanatory models. Sources of mechanistic information include pathway databases, natural language descriptions of mechanisms by human curators, and findings extracted from the literature by text mining. Mechanistic information from multiple sources is de-duplicated, standardized and assembled into sets of mechanistic Statements with associated evidence. Sets of Statements can then be used to assemble both executable rule-based models (using PySB) and a variety of different types of network models. Analytics, Health Informatics, Natural Language Processing, Visualization https://github.com/sorgerlab/indra BSD
Harvard Medical School Bioentities Bioentities is a collection of resources for grounding biological entities from text and describing their hierarchical relationships. Resources were developed by manual curation for improving named entity recognition and grounding in natural language processing and as an ontology for assembly and biological modeling. Analytics, Health Informatics, Natural Language Processing, Visualization https://github.com/sorgerlab/bioentities GPL
Leidos LTR LTR is an early version of Leidos's table reading software. LTR determines which of hundreds of thousands of tables on PMC/PubMed give information about protein interactions, then extracts "index card" information about post-translational modifications, such as phosphorylation, acetylation, and ubiquitination. LTR can read any table on the PMC (PubMed Central) website. (Java) Analytics, Data Formatting, Visualization https://github.com/BigMechanismTableReading/TableReadingSystem Open
Leidos Artemis-S Artemis is Leidos's second-generation table-reading software. Artemis allows for interpretation/reading of a wide variety of target schema types, in addition to the post-translational modifications handled by LTR. Artemis has three components, Artemis-S, Artemis-R, and Artemis-E. Artemis-S takes any table from PMC (generally in HTML, NXML, or Excel), and parses it into a serialized, normalized file that facilitates further processing. (Java) Analytics, Data Formatting, Visualization Not Yet Available Open
Leidos Artemis-R Artemis-R uses special-purpose rules, specified in a YAML file that correspond to the particular schema that a user is targeting. This allows Artemis to read tables that correspond to different schema which the user can define. Artemis-R's input is the set of serialized files output by Artemis-S; its output is a set of tables that conform to the user-defined target schema, as well as a mapping from each table's columns to the target schema. (Java) Analytics, Data Formatting, Visualization Not Yet Available Open
Leidos Artemis-E Artemis-E uses an output-format specification to extract the desired information from tables that have been marked as relevant to a target schema. (Java) Analytics, Data Formatting, Visualization Not Yet Available Open
